# D2 Learning Design Technical Competency Model

## Overview
This document describes the core competencies that define effective Learning Designers. Each competency includes a definition, key behaviors, and examples of application.

---

## 1. Curious Researcher
Definition: Learning designers actively explore contexts, ask questions, and synthesize insights to inform design decisions.

Key Behaviors:
- Conduct stakeholder interviews and user research
- Seek diverse perspectives
- Analyze qualitative and quantitative data
- Identify root causes of performance gaps

Example in Practice:
A designer conducts interviews with learners to understand why a program isnâ€™t driving behavior change and identifies misaligned incentives as the root cause.

---

## 2. Creative Designer
Definition: Designers generate innovative learning solutions using design thinking and interdisciplinary approaches.

Key Behaviors:
- Translate problems into experience concepts
- Generate multiple solution pathways
- Select modalities aligned to learner needs
- Prototype and iterate solutions

Example in Practice:
A designer creates a blended experience combining simulations and peer reflection to improve leadership skills.

---

## 3. Learning Scientist
Definition: Designers apply principles from learning science, behavioral science, and cognitive psychology to maximize effectiveness.

Key Behaviors:
- Manage cognitive load
- Apply spaced learning and retrieval practice
- Align assessments to learning objectives
- Use evidence-based design principles

Example in Practice:
A designer restructures a course to include spaced practice sessions instead of a single workshop.

---

## 4. Collaborative Project Manager
Definition: Designers coordinate stakeholders, manage timelines, and ensure alignment throughout the design process.

Key Behaviors:
- Facilitate alignment conversations
- Manage scope and tradeoffs
- Communicate progress clearly
- Mitigate risks

Example in Practice:
A designer runs weekly alignment sessions with SMEs and stakeholders to keep a project on track.

---

## 5. Influential Storyteller
Definition: Designers communicate ideas and insights in compelling ways to influence decisions and drive engagement.

Key Behaviors:
- Translate insights into narratives
- Frame problems clearly
- Present recommendations persuasively
- Adapt messaging for different audiences

Example in Practice:
A designer reframes research findings into a compelling story that secures leadership buy-in.

---

## 6. Savvy Technologist
Definition: Designers leverage learning technologies to enhance experiences and delivery.

Key Behaviors:
- Evaluate and select tools
- Design across modalities
- Ensure accessibility and usability
- Integrate data and platforms

Example in Practice:
A designer integrates an AI practice tool into a program to enable real-time feedback.

---

## 7. Intentional Learner & Teacher
Definition: Designers actively develop their craft and contribute to the learning of others.

Key Behaviors:
- Reflect on experiences
- Seek feedback
- Share knowledge
- Mentor peers

Example in Practice:
A designer runs internal sessions to share lessons learned from recent projects.
